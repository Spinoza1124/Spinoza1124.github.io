Markdown语法

发表于 2025-04-15 分类于 Markdown ， 编程规范

Markdown 是一种轻量级的标记语言，广泛用于编写文档、博客和技术文章。掌握 Markdown 语法可以帮助你快速创建格式化的文本。可以显著提高文档的可读性和可维护性。

编写规范的 Markdown 文档可以让内容清晰易读，且兼容性良好。以下是编写高质量 Markdown 的实用指南：

1. 基础语法规范

（1）标题层级

  • 前后空行：标题上下留空行，避免粘连
  • 避免跳级：如从 ## 直接到 ####

（2）段落与换行

  • 段落间隔：段落间用空行分隔
  • 强制换行：行尾添加两个空格（多数渲染器支持直接换行）

4. 最佳实践

（1）保持一致性

  • 符号统一：如列表用 - 或 * 保持全文一致
  • 缩进对齐：嵌套内容统一缩进（推荐2或4空格）

（2）语义化写作

  • 避免过度装饰：少用无意义的加粗/斜体

（3）兼容性处理

  • 转义特殊字符：如 #、* 前加反斜杠 \#
  • 测试渲染效果：在目标平台（如GitHub、VS Code）预览

5. 工具推荐

  • 编辑器：

    • markdownlint（自动检查规范）
  • 图床工具：

### **7. 常见错误避免**
- ❌ 标题层级混乱（如从 `#` 直接跳到 `###`）
- ❌ 长段落未分段（超过5行建议拆分）
- ❌ 图片路径失效（建议版本控制中存放到 `/docs/images/`）
- ❌ 使用HTML标签（除非必要，优先用原生Markdown语法）
